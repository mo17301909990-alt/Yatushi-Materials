# 烫金工艺问答策略（客户常见问题与回答方式）

本段供 RAG 检索，指导模型在「型号 + 工艺顺序 / 底材 / 后加工」类问题上的行为；**具体型号是否适用、有哪些系列**必须以对话中注入的【系统实时数据】为准，不得编造。

## 何时必须先追问再答

当用户只给出型号（如 HS#1S、HS#2CO、LLW-438-00、HG#01GR、HS#3CO、SA902-1）并询问「能否先烫金后印刷」「能否烫金后染 UV」「能否过预涂光胶」等，若**未说明烫在什么界面 / 前工序**（例如光粉纸界面、无粉纸、布面纸、已印刷底等），应先**简短追问 1～2 个关键条件**，优先问：**适用界面或前工序是什么？**（烫于什么界面？）以及必要时再问布料纸类型、产品类型、烫金性能类型。追问后再结合【系统实时数据】中的产品字段、兼容规则（hot_stamping_compatibility 等节选）作答；若数据中仍无该型号或规则，明确写「当前系统数据中未提供」，不要虚构。

## 「先烫金后印刷」与型号 HS#1S（示例期望）

问题示例：HS#1S 能用于先烫金后印刷吗？（期望：若上下文不足，AI 会反问烫于什么界面 / 适用界面，以便给出可核验的回答。）回答时：先确认型号是否在【系统实时数据】的产品列表中出现；再查规则节选中与该型号或同系列相关的后工序、兼容矩阵；若无记录，说明缺口并建议用户核对型号书写（大小写、符号）或补充系列、界面、纸类。

## 无粉纸先烫金后印刷要注意什么

关键词：無粉紙、无粉纸、先烫金后印刷、后加工顺序。回答结构建议：分点列出与「无粉纸、烫金附着、后印油墨附着」相关的**一般工艺注意点**（可结合常识），并明确哪些结论必须来自【系统实时数据】中的规则或产品描述；若知识库与实时数据均未展开无粉纸专项条款，应说明「当前片段未收录详细条款」，避免编造客户特定规范。

## HS#2CO 是否有耐磨金纸

关键词：HS#2CO、耐磨、耐磨金纸、金纸类型。必须在【系统实时数据】中检索该型号或同系列产品的「烫金纸张类型」「描述」「规格」等字段是否出现耐磨相关表述；若无，如实说明未提供，可建议用户确认物料编号或系列名。

## LLW-438-00 / HG#01GR 烫金后染 UV

关键词：烫金后染 UV、后染 UV、UV 工艺顺序。先确认型号在数据中是否存在；再查后工序、post_processing、兼容规则中是否允许「烫金 → UV」顺序。若规则未注入或无数值，说明无法从当前上下文确认，可追问适用界面与底涂/油墨体系。

## HS#3CO、布面纸烫金、涤棉 JHT-910：有哪些系列可用与注意事项

关键词：HS#3CO、布面纸、涤棉、JHT-910、系列。期望回答在可能时**列明**：哪些系列或型号「可直接烫」、哪些「需丝印打底油」或类似前处理——**仅当【系统实时数据】或【知识库片段】中有明确字段或规则时**才分条列出；若无，应说明数据未展开到「可直接烫 / 需打底」粒度，并列出已知的系列筛选结果或追问缺失条件（界面、克重、客户编号等）。

## SA902-1 烫金后过预涂光胶

关键词：SA902-1、预涂光胶、过光胶、后加工。检索产品中是否存在该型号及后工序相关字段；结合兼容规则判断是否提及光胶类后加工。无数据则说明缺口。

## 统一提醒（给模型）

涉及 post_processing_steps、后工序、工艺顺序时，优先引用【系统实时数据】中的规则节选与产品属性；表结构段落仅作「有哪些表/字段」说明，不可替代具体业务结论。回答保持简洁、分点，避免冗长罗列数据库表名行数。
